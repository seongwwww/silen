# 실은 — 서비스 기획서

> 서비스명 "실은" (잠정 확정). "실은 아무것도 아니지 않았다."
> 대화로 확정한 결정들을 기반으로 처음부터 새로 정리한 기획서.
> 각 항목은 **[확정]** (함께 결정) / **[권장]** (더블체크 대상) 으로 표시.

---

## 1. 서비스 정의

**한 줄:** 아무 말이나 남기면, 똑같다고 생각한 하루에서 놓친 순간과 변화를 찾아주는 앱. **[확정]**

파는 것은 일기가 아니라 **"내 삶이 생각보다 아무것도 아니지는 않았다는 감각".**

**3층 구조:**
1. 쓸 때 — 가벼운 메모장 (순간·감정을 부담 없이)
2. 하루 끝 — 자동 일기 (조각을 하나의 하루 이야기로)
3. 시간이 쌓이면 — 회고·검색 대화 상대

핵심 엔진은 "차이 발견", 일기는 그 발견을 자연스러운 글로 묶은 최종 결과물.

---

## 2. 타깃 & 문제 [확정]

**첫 타깃:** 기록하고 싶은 마음은 조금 있지만 자기 삶엔 특별한 일이 없다고 여기고, 긴 글은 귀찮은 사람. "오늘 뭐 했어?"에 늘 "그냥 일했지"로 답하는 사람.

**문제:** 이들은 아무것도 안 한 게 아니라 "이 정도는 특별하지 않다"며 스스로 삭제한다. (평소보다 일찍 퇴근 / 처음 들은 노래 반복 / 별거 아닌 성취 / 친구의 다른 말 …)

**핵심 약속:** 입력을 요구하지 말고 **"차이 후보"를 제시**한다. 질문은 "무엇을 했어?"(→"없는데?")가 아니라 **"무엇이 달랐어?"** — 비교 기준을 준다.

---

## 3. 핵심 원칙 [확정]

- **차이 발견이 심장.** 대단한 사건이 아니라 "평소와 달랐던 점"을 찾는다.
- **탐지는 통계가, 서술은 AI가.** LLM에 "차이를 찾아줘" 금지(창작·과잉해석). 통계로 탐지 → 검증된 것만 LLM이 담백하게 문장화.
- **의무가 아니라 선물.** 안 남기면 아무 일도 안 일어남. 죄책감 유발(스트릭 압박·빨간 배지) 금지. 똑같은 날은 똑같았다고 남겨도 됨.
- **진실성.** 문체·길이는 자유, 사실은 고정. 있는 내용을 감정까지 담백하게, 없는 사건 창작 금지.
- **과잉해석 금지.** "새로운 걸 하세요"류 자기계발 앱화 금지. 이미 있었던 작은 변화를 발견해줄 뿐. "오늘도 비슷했어요. 그래도 어제보다 조금 일찍 잤네요."
- **원본/AI생성 분리.** 근거 추적 가능하게. 모든 결과에 "왜 이걸 찾았는지" 표시.
- **한국 일상 특화가 해자.** 카톡 스샷·인스타 저장·한국어 OCR·한국 지명.

## 5. 비교(baseline) 엔진 [확정, 세부 권장]

"이 유저의 평소"를 모델링하고 이탈=차이를 찾아 억지 없이 돌려준다.

**평소 = 여러 축:** 수치·시간축(퇴근/취침·걸음·스크린타임: 이동평균+요일계절성) / 범주축(집밥vs외식·게임vs산책·만난 사람: 빈도) / 감정축(요일별 기분·활동↔기분).

**두 종류의 차이:** 오늘 vs 평소("오늘의 다른 점") / 시간을 가로지른 패턴("친구와 대화한 날엔 기분이 빨리 좋아졌다" = 내가 몰랐던 나, 리텐션 핵심).

**탐지→서술 분리:** z-score·first-occurrence·빈도 이탈로 탐지 → 검증된 차이만 LLM이 담백하게 서술. 탐지 없으면 억지로 안 만듦.

**의미 필터:** 통계적 이탈 ≠ 이야기. 사람이 읽고 느끼는 차이만. (핏빗 통계 덤프 아님)

**콜드스타트 = 온보딩 서사:** 1~2주 "네 평소 배우는 중" → 7일차에 첫 리포트.

**계층 산출물:** 오늘의 다른 점 / 별거 아닌 성취(미뤄둔→닫힘) / 감정이 바뀐 순간 / 오늘의 한 문장 / 자동 일기.

**피드백 루프:** "차이 후보" 확인/기각 = 라벨 → baseline·필터 개인화.

**후속 질문 규칙(과요청 방지):** 감정 강함 / 평소와 다름 / 새 사람·장소·활동 / 이전 기록 후속 / 일기 완성에 정보 필요 — 이럴 때만 한 번 더, 한두 번 터치로.

## 6. 일기 생성 루프 [확정]

- 낮: 순간을 가볍게 메모(텍스트 기본 + 사진/음성/위치 선택), 분류·태그 없음
- 정해둔 시간: 그날 메모로 초안 자동 생성 → 알림 1건
- 메모 0개면 아무 일 없음, 1~2개면 짧게, 3개+면 시간·감정 흐름 엮기
- 톤 조절 3층: ①기본 설정(매일 자동) ②생성 전 주문 ③생성 후 대화형 수정. 프리셋 칩 + 자유 입력
- 초안 → 인라인 수정 or 바로 확정. 안 건드리면 draft 보존
- 늦은 메모 → "다시 만들기" 1회(자동 재생성 X)
- 원본 메모와 확정 일기 분리 저장

## 7. 반자동 캡처 + 프라이버시

**캡처 소스(전부 opt-in, 기본 OFF):**
- MVP: **걸음** + 수동 기록(텍스트·사진). 걸음은 권한 가볍고 배터리 무해 **[확정]**
- 이후: **방문 장소(백그라운드 위치)** → 차이 재료로 강력하나 권한 거부율·배터리·심사 부담 커서 확장. 그 뒤 음악·일정·결제·스크린타임·게임

**권한 요구 시점 [확정]:** 센서 권한은 Day 0에 안 물음(온보딩 가벼움 유지). 며칠 써서 가치를 느낀 뒤 "걸음도 켜면 이런 것도 찾아줘요"로 유도.

**동의 설계 [확정]:** 소스별 개별 토글, 각 토글이 "무엇을 읽고/만들고/저장하는지" 설명. 켜기 전엔 안 읽음. 센서 데이터는 조용히 저장 X → "차이 후보"로 제시, 유저 확인 시 기록화. **파생 신호만 저장(원자료 최소화)** — 연속 GPS 대신 "방문 장소 라벨". **[권장]**

**일기 보호 장치 [확정]:** 원본/AI 분리 · 기억별 잠금(검색·일기·회고·탐지에서 제외 선택) · 민감 기록 검색 제외 · 데이터 내보내기 · 삭제 시 원본+임베딩+색인 함께 삭제 · 센서 해제 시 파생 삭제 · AI 학습에 유저 데이터 미사용 · 위치 저장 여부 선택 · 자동 녹음 MVP 제외 · 처리 출처 표시.

**법·타인 정보 [권장]:** PIPA 준수. 사진 속 타인은 최소 수집, MVP 얼굴 인식 안 함. 온디바이스는 MVP 클라우드, 후일 프라이버시 프리미엄.

## 8. 7일 실험 온보딩 + 공유 카드 [확정]

**7일 실험 = 제품의 주 유입 장치.** 광고·랜딩·공유 카피 모두 "7일 뒤, 당신이 몰랐던 반복·변화 3가지"로.

- Day 0: 첫 기록 5초 성공 체험 + 일기 시간·톤 프리셋 설정. **센서 권한은 안 물음**
- Day 1~6: 가벼운 "차이 후보" 넛지(끌 수 있음) + 밤마다 "오늘의 다른 점" 한 줄
- Day 7: **"당신이 몰랐던 이번 주"** 리포트(baseline 첫 산출) = aha moment
- **Day 7은 끝이 아니라 시작으로 연결:** "7일간 이만큼 발견했어요. 다음 주엔 더 정확해져요" — baseline이 계속 좋아진다는 걸 보여주며 이탈 방지

**공유 카드:** 일기 원문 X, 요약 카드만("이번 주의 나 — 가장 많이 한 것/기분 좋았던 순간/처음 한 것/별거 아닌 성취"). **기본 비공개 + 공유 전 항목별 편집·제외 + 미리보기.** 바이럴 = 발견한 자기 모습.

## 9. 회고 대화 & 검색 [확정]

- 개인 기록 위 RAG. baseline 산물 재사용
- 실제 기록만 인용 + 근거 카드. 관찰 제시, 단정 X
- **회고형 그거뭐였지**("작년 이맘때 고민?")는 baseline 지으면 공짜로 딸려옴
- **아카이브형 그거뭐였지**(저장한 스샷·링크 찾기)는 공유하기/스샷 캡처만 먹여주면 됨 (확장)
- 검색: 벡터+키워드/OCR 하이브리드 + 필터(날짜·장소·사람·출처) + 재정렬 + 원본 카드

## 13. 구현 순서 & 8주 계획 [권장]

**순서:** 데이터모델→인증→텍스트기록+감정태그→baseline+차이탐지→계층산출물+서술→일기루프+톤3층→스케줄+알림→7일온보딩+리포트→임베딩+회고검색/대화→소수센서+동의UI→사진→(이후)아카이브·Drive.

- W1 데이터모델·인증·텍스트기록·감정태그 + eval 뼈대
- W2 baseline 엔진 + 차이 탐지 + 탐지→서술 분리
- W3 계층 산출물 + 일기 생성 + 톤 3층
- W4 스케줄·알림 + 초안/수정/확정
- W5 7일 온보딩 + 7일 리포트
- W6 임베딩 + 회고 검색/대화 + 근거 카드
- W7 걸음·장소 센서 + 동의 UI + 사진 + 프라이버시(잠금·삭제·내보내기)
- W8 공유 카드 + 가드레일 튜닝 + eval 회귀 + 베타

**현실 메모:** 3층(회고 대화 포함)까지 MVP에 넣으므로 8주는 빡빡함 → **버퍼 1주(총 9주) 권장**. 회고 대화는 데이터가 쌓여야 진가가 나오므로 출시 시엔 얇게, 이후 자연히 개선.
