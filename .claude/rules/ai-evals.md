# AI · Eval 규칙

핵심: **탐지=통계, 서술=LLM.** 프롬프트 상세는 `docs/design/prompts-draft.md`.

## 강제 사항

- 차이 탐지 결과는 **deterministic detector에서만** 생성. LLM을 탐지에 쓰지 않는다.
- LLM 입력에 **evidence ID(memory/difference)** 포함.
- LLM 출력은 **구조화 스키마(Structured Output)로 검증.** 스키마 밖 텍스트 거부.
- **근거 없는 문장 비율 0% 목표.** 출력의 모든 고유 사실이 입력 facts에 존재하는지 후검증.
- 조언·자기계발·응원 문구, 통계에 없는 인과("~때문에") 혼입 0.
- 감정 강도가 원본 태그를 넘지 않음. 톤 변경 시 사실 집합 불변(문체만).

## 골든셋 & 회귀

- eval 골든셋을 **1일차부터** 유지(`evals/`). 각 케이스 = 입력 fixture + 기대 속성.
- **false positive 회귀 차단** — 골든셋 통과가 CI 게이트.
- **프롬프트·detector·모델 변경 시 eval 자동 실행**(Hook 권장).
- 필수 케이스: 빈 날(억지 생성 X) · 미세 이탈(의미필터 탈락) · 환각 유혹 · 감정 승격 · 톤 불변성 · 근거 정합(used_ids ⊆ 입력) · 상관≠인과 · 회고 미존재.
- **"평범한 날을 억지로 특별하게 만들지 않는" 테스트**를 반드시 포함.

## 측정 지표

- 환각율 / 조언혼입율 / 톤불변성 통과율 / 근거정합율 / false positive율. 릴리스 전 리포트.

## 비용

- 탐지는 통계로(무료). LLM은 lazy 서술 호출에만. 필터·게이트는 소형 모델. 전역 시스템 프롬프트는 캐시.
