# 프론트엔드 규칙

스택: Next.js(App Router) · TypeScript · Tailwind · shadcn/ui · 모바일 우선 PWA.
공식 `frontend-design` Skill과 함께 쓰되, 아래는 **실은 전용 제약**으로 항상 우선한다.

## 제품 톤 (필수)

- **죄책감 유도 UI 금지.** 스트릭, 연속 기록 카운터, 빨간 배지, "N일째 안 썼어요" 류 압박 금지.
- 빈 상태는 실패가 아니다. "오늘은 조용했네요" 처럼 담담하게. 억지 독려 금지.
- **"AI가 발견했다"고 말하지 않는다.** 근거가 있는 차이만, "왜 찾았는지"를 사람 말로 함께 표시.
- 차이 카드의 **[맞아요] / [아니에요] 버튼을 명확히 구분**한다(색·위치·라벨). 오탭 유발 배치 금지.
- **생성된 일기와 원본 기록을 시각적으로 구분**한다(배경/라벨/아이콘). 무엇이 AI 산물인지 항상 알 수 있어야 한다.

## 상태 처리 (모두 구현)

- 모든 데이터 화면은 **empty / loading / processing / error / offline** 상태를 명시적으로 구현한다.
- AI 생성(일기·차이)은 비동기 → **processing 상태**를 반드시 노출. 낙관적 UI로 없는 결과를 보여주지 않는다.
- 실패는 조용히 삼키지 않는다. 재시도 경로를 제공하되 자동 무한 재시도 금지.

## 프라이버시 UX

- 동의 토글은 소스별 개별. 각 토글이 **"무엇을 읽고/만들고/저장하는지"** 한 줄로 설명.
- **동의 철회·전체 삭제**는 결과를 구체적으로 안내("원본·임베딩·검색 색인·파생 신호가 모두 삭제됩니다"). 되돌릴 수 없으면 명시.
- 잠근(locked) 기억은 검색·일기·회고·탐지에서 제외됨을 UI에서 드러낸다.

## 접근성·품질 (frontend-design로 점검)

- 타이포·색 대비(WCAG AA)·포커스 표시·키보드 내비게이션·`prefers-reduced-motion` 대응.
- 반응형: 모바일 우선, 터치 타깃 44px 이상.
- 색만으로 의미 전달 금지(맞아요/아니에요에 라벨·아이콘 병행).

## 코드 컨벤션

- 서버 컴포넌트 기본, 클라이언트 컴포넌트는 `"use client"` 최소화.
- 데이터 패칭은 서버에서. 클라이언트 상태는 필요한 만큼만.
- Tailwind 코어 유틸 사용, 매직 넘버 대신 디자인 토큰(spacing/color scale).
- 컴포넌트는 단일 책임. props 타입 명시, 기본값 제공.
