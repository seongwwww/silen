# 백엔드 규칙

앱·API = Next.js Route Handlers. 차이 탐지·AI 잡 = **Python 워커**(numpy/scipy/pandas + LLM/임베딩). 둘은 비동기 큐로 연결.

## 경계 (LLM 오염 방지)

- **탐지 코드에 LLM 호출 금지.** detector는 순수 통계/규칙(z-score·first-occurrence·freq-shift·상관)만.
- LLM은 서술 단계에서만. 입력에 evidence(memory_id/difference_id) 포함, 출력은 구조화 스키마로 검증.
- LLM 출력의 모든 고유 사실이 입력 facts에 존재하는지 후검증(가드레일)을 통과해야 저장.

## 멱등성·큐

- **모든 큐 작업은 멱등.** (user_id, date, job_type) 등 자연키로 upsert. 재시도가 중복 레코드를 만들면 안 된다.
- 일기 생성은 하루 1건 보장(유니크 제약 + upsert). 늦은 메모 → 명시적 "다시 만들기"만, 자동 재생성 금지.
- 작업 상태(queued/processing/done/failed)를 기록. 실패는 재시도 횟수 상한 + 데드레터.

## 시간·타임존

- "하루"의 정의 = **사용자 로컬 자정 기준**. 경계 계산은 단일 유틸(`lib/time`)에서만.
- 저장은 UTC(timestamptz), 표시·집계는 사용자 타임존으로 변환. 전날 메모가 오늘 일기에 새지 않도록 경계 테스트 필수.

## API 규약

- REST, 명사 복수형. 인증은 미들웨어에서 일괄. 모든 핸들러는 user 스코프 강제.
- 입력 검증(zod 등) → 실패 시 4xx + 구조화 에러. 서버 에러는 5xx + 상관관계 ID.
- 페이지네이션·정렬·필터는 쿼리 파라미터. 큰 결과는 커서 기반.

## 로깅·관측성

- **사용자 기록 본문·일기 텍스트를 로그/APM/에러 추적에 남기지 않는다.** ID·메타데이터·카운트만.
- 구조화 로깅(JSON), 상관관계 ID로 요청↔워커 잡 연결.
- PII를 스택트레이스·브레드크럼에 싣지 않도록 스크러빙.

## 에러 처리

- 실패를 삼키지 않는다. 복구 가능/불가능을 구분해 처리.
- 외부 호출(LLM·STT·임베딩)은 타임아웃·재시도(지수 백오프)·서킷브레이커. 비용 상한 인지(lazy 처리).
