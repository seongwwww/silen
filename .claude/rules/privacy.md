# 프라이버시 규칙

PIPA 준수. 개인 기록이 핵심 자산이므로 프라이버시는 기능이 아니라 제약이다.

## 수집·동의

- 센서·사진·위치는 **opt-in, 기본 OFF.** 켜기 전엔 읽지 않는다.
- 소스별 개별 동의(consents 테이블). 각 동의는 범위(scope)와 무엇을 저장하는지 기록.
- **파생 신호만 저장** — 연속 GPS 대신 방문 장소 라벨, 원자료 최소화.
- 사진 속 타인 정보 최소 수집. MVP 얼굴 인식 안 함.

## 삭제·철회 (완전성 보장)

- 기억 삭제 = **원본 + assets(Storage) + embeddings + 검색 인덱스 + 파생 신호** 모두 삭제.
- 동의 철회 시 해당 소스의 파생 데이터 삭제.
- 삭제는 deletion ledger로 단계 추적(database.md 참조). 부분 실패 시 재개 가능해야 한다.
- 삭제·철회 결과를 사용자에게 구체적으로 안내(무엇이 지워지는지, 되돌릴 수 없는지).

## RAG·검색

- 검색·회고는 **본인 데이터만.** 모든 벡터 질의에 user_id 선필터. 교차 사용자 노출은 심각 결함.
- locked/민감 기억은 검색·회고·탐지에서 제외.
- 회고 답변은 실제 기록만 인용, 근거 카드 제공, 단정 금지.

## 로깅·학습

- 기록 본문·일기 텍스트를 로그·APM·에러 추적에 남기지 않는다(backend.md와 동일).
- 사용자 데이터를 모델 학습에 사용하지 않는다.

## 변경 시 필수 검사

Auth·삭제·RAG·동의 관련 코드를 바꾸면 `/security-review` 를 반드시 실행하고, 교차 사용자 접근·삭제 누락·본문 로깅을 중점 점검한다.
